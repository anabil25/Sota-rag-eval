101-4          RECEIPT OF AN IDENTIFIABLE APPLICATION

Every DPADivision of Public Assistance office must accept any identifiable application that is presented. An identifiable application is one that contains a name, address, and signature. The signature can be anywhere on the form.

Refer to the policy manuals for specific information regarding the application signature requirements for each program.

101-4 A.      APPLICATION FORM WITH NO SIGNATURE

If an application is received which is not identifiable, it should be returned to the individual immediately, when possible. Do not register the application.

101-4 B.      IDENTIFIABLE APPLICATION - STATEMENT OF TRUTH NOT SIGNED

If an identifiable application is received and the Acknowledgement of Understanding and the Statement of Truth page on the application form (GEN 50CApplication for Services) is not signed, register the application as a request for service for selected programs ([MAGI](#)Modified Adjusted Gross Income Medicaid, [HAP](#)Heating Assistance Program, and [SB](#)Senior Benefits must be registered as normal) and a copy of the application returned to the household for signature.

An interview should be conducted and the application pended for the signed Acknowledgement of Understanding and Statement of Truth page, and any additional verification needed to make an eligibility determination. Do not make an eligibility determination until the Acknowledgement of Understanding and State of Truth page is signed.

Note:

Acknowledgement of Understanding is part of the Statement of Truth on page 18 of the GEN 50CApplication for Services application.

An eligibility determination cannot be made without a completed, signed application.

Scan the application into ILINX and return the original to the household with instructions to complete and return the form to [DPA](#)Division of Public Assistance before the application can be processed. Enter a [CANO](#)Case Notes stating that the form was returned to household for completion.

101-4 C.      IDENTIFIABLE APPLICATION WITH NO PROGRAM SELECTED

If an identifiable application is received and no program has been checked by the client, the agency must make every effort to contact the client to confirm which program is being requested. If the client cannot be reached, the application must be registered as a request for [SNAP](#)Supplemental Nutrition Assistance Program (RF) and pended for the necessary verification needed to make an eligibility determination.

A [CANO](#)Case Notes should be created documenting the contact or attempted contact with the household. The application is scanned into ILINX.
